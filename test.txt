 Hello git
$ git init - ініціалізація локального репозиторію
$ git add - добавити файли в коміт кандидат
$ git commit - закомітати
$ git status - статус локального репозиторію
$ git log --pretty=format:'%h %ad | %s%d | %an' --date=short - history in nice format
$ git checkout (commit hash) - to navigate to any commit by hash
$ git checkout (branch(master)) - to navigate back to master branch from any commit
$ git reset HEAD (file name) - зробити unstage (cancel add) потім git checkout (file name) поверне стан файлу до останнього закоміченого
Если мы выполним команду git reset --hard a1e8fb5, история коммитов будет сброшена до указанного коммита. для локальних змін
$ git revert HEAD --no-edit - відмінити останній коміт - для публічних змін
$ git revert <commit hash> --no-edit - відмінити всі коміти до даного
Шоб пофіксити конфлікт при реверті чи мерджі відкриваємо конфліктні файли редагуємо їх як нам потрібно робимо ед і
 git revert\merge --continue і commit
$ git branch - подивитись бренчі і на якій ми зараз
$ git checkout -b new_branch - створити нову гілку і перейти на неї
----------
Merge steps
-----------
$ git checkout <branch name to which you want to merge changes>
$ git merge <branch name from which you want to merge changes>
